laboratorio 3

Volvamos a nuestra lista de contactos. Después de algunos ajustes recientes (es decir, usar una matriz y objetos), en realidad está empezando a parecerse a una lista. Sin embargo, persistía un problema bastante grave. Los cambios de datos en la lista, como agregar un nuevo contacto, deben especificarse de inmediato en el código del programa. ¿Qué pasa si queremos darle al usuario la posibilidad de ingresar los datos del contacto agregado mientras el programa se está ejecutando? Modifique el programa para agregar, al final de la lista, no el contacto que se proporciona en el código, sino el que el usuario proporcionará durante la ejecución del programa. Utilizar elinmediatométodo para hacer esto. Al final, muestra el primer y último contacto de la lista.

laboratorio 4

Durante la última modificación del programa con la lista de contactos, permitimos al usuario agregar un nuevo contacto con los datos ingresados ​​por el usuario mientras se ejecuta el programa. Vayamos un paso más allá: intentemos modificar el programa para que el usuario pueda elegir qué quiere hacer con la lista. Tendrán la opción de: mostrando el primer contacto (primero) mostrando el último contacto (último) agregando un nuevo contacto (nuevo) Al agregar un nuevo contacto, verifique si el usuario ha ingresado todos los datos necesarios.
