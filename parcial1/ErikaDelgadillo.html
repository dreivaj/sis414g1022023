LABORATORIO 1


Volvamos a nuestra lista de contactos. Después de algunos ajustes recientes (es decir, usar una matriz y objetos) en realidad está comenzando a verse como una lista. Sin embargo, sigue existiendo un problema bastante grave. Los datos cambiantes en la lista, como agregar un nuevo contacto, deben proporcionarse de inmediato en el código del programa. ¿Y si queremos dar al usuario la posibilidad de introducir los datos del contacto añadido mientras se ejecuta el programa? Modifique el programa para agregar, al final de la lista, no el contacto, que se da en el código, sino el que el usuario dará durante la ejecución del programa. Utilice el método prompt para hacer esto. Al final, muestra el primer y el último contacto de la lista.


LABORATORIO 2


Durante la última modificación del programa con la lista de contactos, permitimos que el usuario agregue un nuevo contacto con los datos ingresados por el usuario mientras se ejecuta el programa. Vayamos un paso más allá: intente modificar el programa para que el usuario tenga la opción de elegir lo que quiere hacer con la lista. Podrán elegir entre: Mostrando el primer contacto (primero) Mostrar el último contacto (último) Agregar un nuevo contacto (nuevo) Al agregar un nuevo contacto, verifique si el usuario ha ingresado todos los datos necesarios. Si falta al menos uno de los tres valores (nombre, teléfono o correo electrónico), no agregue el contacto.
